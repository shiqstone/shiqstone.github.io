简单理解Typescript里的泛型

March 1, 2023

您可能认为 TypeScript 中的泛型（generics）实际上是三个独立的概念：

  • 将类型传递给类型
  • 将类型传递给函数
  • 从传递给函数的参数推断类型
// 1. Passing types to types
type PartialUser = Partial<{id: string; name: string}>;

// 2. Passing types to functions
const stringSet = new Set<string>();

// 3. Inferring types from arguments passed to functions
const objKeys = <T extends object>(obj: T) => {
    return Ojbect.keys(obj) as Array<keyof T>;
};

const keys = objKeys({a: 1, b: 2});
// ^? const keys: ("a" | "b")[]

让我们从将类型传递给类型开始。

在 TypeScript 中，您可以声明一个表示对象、基元、函数的类型——任何您想要的类型。

export type User = {
  id: string;
  name: string;
};

export type ToString = (input: any) => string;

但是假设您需要创建一些具有相似结构的类型。 例如，数据形状。

下面的代码不是很符合DRY —— 我们可以清理它吗？

type ErrorShape = {
  message: string;
  code: number;
};

type GetUserData = {
  data: {
    id: string;
    name: string;
  };
  error?: ErrorShape;
};

type GetPostsData = {
  data: {
    id: string;
    title: string;
  }[];
  error?: ErrorShape;
};

type GetCommentsData = {
  data: {
    id: string;
    content: string;
  }[];
  error?: ErrorShape;
};

如果您倾向于 OOP，则可以使用可重用的界面来做到这一点…

interface DataBaseInterface {
  error?: ErrorShape;
}

interface GetUserData extends DataBaseInterface {
  data: {
    id: string;
    name: string;
  };
}

interface GetPostsData extends DataBaseInterface {
  data: {
    id: string;
    title: string;
  }[];
}

interface GetCommentsData extends DataBaseInterface {
  data: {
    id: string;
    content: string;
  }[];
}

但是创建一个“类型函数”更简洁，它接受数据类型并返回新的数据形状。

// Our new type function!
type DataShape<TData> = {
  data: TData;
  error?: ErrorShape;
};

type GetUserData = DataShape<{
  id: string;
  name: string;
}>;

type GetPostsData = DataShape<
  {
    id: string;
    title: string;
  }[]
>;

type GetCommentsData = DataShape<
  {
    id: string;
    content: string;
  }[]
>;

理解这个语法很重要，因为它会在后面出现。

尖括号 <TData> 告诉 TypeScript 我们要向该类型添加一个type argument。

我们可以给 TData 起任何名字，它就像一个函数的参数。

这是一个通用类型。

// Generic type
type DataShape<TData> = {
  data: TData;
  error?: ErrorShape;
};

// Passing our generic type
// another type
type GetPostsData = DataShape<
  {
    id: string;
    title: string;
  }[]
>;

通用类型可以接受多个类型参数。您可以为类型参数提供默认值。您甚至可以对类型参数提供约束，以便只能传递某些类型。 但是，现在我们先继续之前的话题。

如果我告诉您,可以将类型传递给的不只是types，您会怎么办？

const createSet = <T>() => {
  return new Set<T>();
};

const stringSet = createSet<string>();
// ^? const stringSet : Set<string>

const numberSet = createSet<number>();
// ^? const numberSet : Set<number>

没错——就像类型可以接受类型作为参数一样，函数也可以。

在本例中，我们在声明 createSet 时在括号前添加了一个 <T>。

然后我们将该 <T> 手动传递给 Set()，它本身允许您传递类型参数。

export const createSet = <T>() => {
  return new Set<T>();
};

这意味着当我们调用它时，我们可以将 的类型参数传递给 createSet。 我们最终得到了一个只能传递字符串的集合。

const stringSet = createSet<string>();

// Argument of type 'number' is not assignable
// to parameter of type 'string'.
stringSet.add(123);

这是谈论泛型时的第二个意思——手动将类型传递给函数。

如果你使用 React，你就会看到这一点，你需要将类型参数传递给 useState。

const [index, setIndex] = useState<number>(0);

但是，您还会注意到 React 中的另一种行为。 也就是说，在某些情况下，您不需要传递类型参数来推断它…

const [index, setIndex] = useState(0);
//     ^? const index: number

// WHAT IS THIS SORCERY

让我们再看看我们的 createSet 函数。 您会注意到它不接收actual 参数, 仅接收类型参数。

const createSet = <T>() => {
  return new Set<T>();
};

但是如果我们改变我们的函数让它接受集合的初始值呢？ 我们知道初始值需要与 Set 的类型相同，所以我们将其输入 T。

const createSet = <T>(initial: T) => {
  return new Set<T>([initial]);
};

现在，当我们调用它时，我们需要传入一个初始值，并且它需要与我们传递给 createSet 的类型参数具有相同的类型。

const stringSet = createSet<string>("matt");


// Argument of type 'string' is not assignable
// to parameter of type 'number'.
const numberSet = createSet<number>("pocock");

但这是神奇的事情。 TypeScript 可以从'initial'推断出'T'的类型。 换句话说，类型参数将从运行时参数推断出来。

const stringSet = createSet("matt");
//    ^? const stringSet: Set<string>

const numberSet = createSet(123);
//    ^? const numberSer: Set<number>

如果您使用IDE，可以通过将鼠标悬停在其中一个 createSet 调用上来检查这一点。 当我们向它传递一个字符串时，您会看到正在推断 <string> 。

const createSet: <string>(initial: string) => Set<string>

在 useState 中也是如此 - 请参阅工具提示中的 useState<number> 语法。

在我显示在该线程最顶部的 objKeys 函数中。 这个也有一些额外的好处：

  • 我们将 T 限制为一个对象，以便它可以传递给 Object.keys（它只接受对象）
  • 我们强制 Object.keys 的返回类型为 Array<keyof T>
const objKeys = <T extends object>(obj: T) => {
  return Object.keys(obj) as Array<keyof T>;
};

总的来说，您认为的“泛型”实际上是几种不同的模式：

  • 将类型传递给类型 - DataShape
  • 将类型传递给函数 - createSet()
  • 从传递给函数的参数推断类型 - useState(0)
